# Project Workflow

## Development Process
- **Task Granularity**: Break down work into small, verifiable tasks.
- **Testing**: Minimum 80% code coverage for core logic.
- **Commits**: Commit changes after every completed task.
- **Documentation**: Update `plan.md` and `spec.md` as the project evolves.

## Phase Completion Protocol
1.  Verify all tasks in the phase are completed.
2.  Run full test suite.
3.  Perform manual verification of the UI/UX.
4.  Checkpointing: Commit and tag the project state.
